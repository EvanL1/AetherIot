# 平台状态与路线图

本站分别报告已经实现、实验性、部分实现和规划中的能力。产品名称和愿景不会自动提高技术成熟度。

## AI 原生最终用户体验

**已经实现的基础：** 面向智能体的 Markdown 与索引、Agent Skill、运行时能力发现、OpenAPI、受治理应用命令、AetherEdge MCP 工具与资源、确定性本地规则、审计证据，以及公开契约与一致性仓库。

**实验性或部分实现：** AetherCloud 的传输中立 MCP 应用接口、期望/上报/应用部署、受治理作业、CloudLink、遥测持久化，以及 Edge 与 Cloud 的开发验证。

**规划中：** 家庭或站点语义上下文、最终用户对话智能体、类型化意图/方案/策略契约、意图到自动化编译器、历史仿真、按需生成的确认体验、临时行为有效期、效果评估和持续受治理调整。当前任何发行版都不能把这些能力描述成已经完成的产品。

## AetherEdge

**已经实现：** 六服务运行时、SHM 实时状态权威、嵌入式本地运行、受治理命令、`aether` CLI、`aether-edge-sdk`、Pack v1、MCP 与 OpenAPI 基础，以及已签名的 `v0.5.0` 源码、运行时和 CLI 制品。

**实验性：** CloudLink MQTT v1 边端基础、由应用确认驱动的持久队列、AetherContracts alpha.3 消费，以及真实 Broker 开发证据。

**规划或受门槛限制：** 生产 CloudLink 密钥生命周期、签名确认、完整联合一致性、旧版切换和剩余应用边界迁移。

## AetherCloud

**已经实现的基础：** 模块化单体领域与应用切片、能力驱动的提供方适配、仅规划 OpenTofu、网关注册、部分 CloudLink 与遥测持久化、制品/部署/作业基础、审计与集成切片、可观测性，以及传输中立的 MCP 应用接口。

**实验性或部分实现：** MQTT 编解码与入口、本地和 AWS IoT 开发验证、PostgreSQL 已接受遥测确认发件箱，以及有限的审计接口。

**规划或受门槛限制：** 生产身份、完整 CloudLink 持久性与映射、生产组合和工作进程、公开作业与部署交付、强化的外发集成，以及可连接的 MCP 服务器。

## AetherContracts

**已经实现但仍属实验性：** alpha.3 规范、封闭 Schema、测试夹具、TCK、摘要锁定的消费者验证，以及四种语言的测试夹具绑定。

**规划或受门槛限制：** 生产认证密钥生命周期、签名持久确认、完整生产编解码器，以及可用于生产 CloudLink 切换的发行版。

## 平台文档

**已经实现：** 统一中英文文档入口、产品概览、AI 原生平台说明、部署拓扑、用户旅程、端到端联动指南、兼容性矩阵、状态页面和 AetherIot 到 AetherEdge 迁移指南。

**规划中：** 自动聚合三个仓库的版本状态、发行通道状态源，以及在地址合适时建立未来的 GitHub 组织。
